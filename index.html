📊 DATA FORMATS

Understanding how information is structured and stored

A data format

A data format is the way data is structured, organized, and stored so that computers and people can read, process, and exchange information correctly.

Example
  • As text: "20"
  • As a number: 20
  • Inside JSON: "age": 20
Note

Each representation follows a different data format.

Importance of Data Formats

  • Enable communication between systems (e.g., mobile app → web server via JSON).
  • Ensure data accuracy and consistency (e.g., dates as YYYY-MM-DD).
  • Improve efficiency and performance (binary loads faster than text).
  • Support data storage and retrieval (databases rely on proper formats).
  • Make data understandable to humans (text-based formats are readable/editable).

Classification

Structured Data

DEFINITION:
Structured data follows a fixed and organized structure, usually in rows and columns.

Characteristics:

  • Predefined format
  • Easy to search and analyze
  • Stored in databases

Examples: SQL databases, Excel tables, CSV files

Uses: Banking systems, school records, inventory systems

Semi-Structured Data

DEFINITION:
Semi-structured data has some structure, but it is flexible and does not follow strict tables.

Characteristics:

  • Uses tags or key–value pairs
  • Flexible structure
  • Easy to share across systems

Examples: JSON, XML, YAML

Uses: Web APIs, mobile apps, configuration files

Unstructured Data

DEFINITION:
Unstructured data has no predefined format or structure.

Characteristics:

  • Difficult to organize
  • Requires special processing
  • Large in volume

Examples: Images, videos, audio, social posts, emails

Uses: Social platforms, AI/ML, multimedia systems

Advantages vs Disadvantages

Advantage
  • Organizes data clearly
  • Makes data easy to share
  • Improves system compatibility
  • Supports automation and AI
  • Reduces data errors
Disadvantage
  • Some formats are complex
  • Large file sizes (text formats)
  • Not all formats are human-readable
  • Wrong format choice causes inefficiency
  • Requires conversion between formats

📁 Common Data Formats

Text, Images, Audio & Video formats explained

Data formats are used to store and organize digital information. Common categories: text, images, audio, and video. Different formats are suited for different uses and devices.

TEXT FORMATS: TXT & CSV

TXT (Plain Text Files)
  • Stores only simple text with no formatting
  • Extremely small and compatible with all systems
  • Used for notes, logs, and documentation
CSV (Comma-Separated Values)
  • Stores data in table or spreadsheet format
  • Columns separated by commas
  • Common in Excel, databases, and data analysis

TEXT FORMATS: JSON & XML

JSON (JavaScript Object Notation)
  • Stores structured data in key–value pairs
  • Lightweight and human-readable
  • Used in web apps and APIs
XML (Extensible Markup Language)
  • Uses tags to describe and structure data
  • Flexible but more complex and verbose
  • Common in enterprise systems and configurations

TEXT FORMAT: YAML

YAML
  • Human-friendly structured text format
  • Designed for configuration and automation
  • Popular in DevOps and cloud technologies

Comparing Text Formats

  • TXT — simple unformatted text
  • CSV — table-style structured data
  • JSON & XML — complex structured data
  • YAML — readable configuration data

IMAGE FORMATS: JPG & PNG

JPG / JPEG
  • Best for photos and realistic images
  • Uses lossy compression to reduce size
  • Very common in cameras and websites
PNG
  • Lossless compression keeps full quality
  • Supports transparency
  • Ideal for logos, icons, and graphics

IMAGE FORMATS: GIF & SVG

GIF
  • Supports simple animations
  • Limited to 256 colors
  • Popular for memes and reaction images
SVG
  • Vector-based graphics
  • Can scale without losing quality
  • Used for icons, logos, and diagrams

AUDIO FORMATS: MP3 & WAV

MP3
  • Compressed audio format
  • Small file size with good quality
  • Common for music and streaming
WAV
  • Uncompressed high-quality audio
  • Very large file sizes
  • Used in studios and professional recording

AUDIO FORMAT: AAC

AAC
  • Advanced compressed audio format
  • Better quality than MP3 at the same file size
  • Used by YouTube, Apple Music, and streaming platforms

VIDEO FORMATS: MP4 & AVI

MP4
  • Most widely supported video format
  • Good quality with efficient compression
  • Works on almost all devices and platforms
AVI
  • Older Microsoft video format
  • Larger file sizes
  • Still used in editing and archiving

VIDEO FORMAT: MKV

MKV
  • Flexible video container format
  • Supports multiple subtitles and audio tracks
  • Popular for high-quality media and movies

⚡ Binary Data Formats

Efficient data storage in raw bytes

Binary Files

Technical Characteristics
  • Stored as raw bytes (0s and 1s)
  • Not human-readable without tools
  • Platform-dependent if not designed carefully
  • Efficient for large data and real-time processing
Real-World Usage
  • Images (PNG, JPEG, BMP)
  • Audio (MP3, WAV)
  • Video (MP4, AVI)
  • Executable programs (.exe, .bin)
  • Game assets and simulations

Protocol Buffers (Protobuf)

Key Features
  • Compact binary representation
  • Faster than JSON or XML
  • Strongly typed schema (.proto files)
  • Backward/forward compatibility
Real-World Applications
  • Microservices communication
  • gRPC APIs
  • Distributed systems
  • Cloud services
  • Mobile applications

SQL (Structured) vs NoSQL (JSON-Based)

SQL (Structured Data)
  • Schema-based, relational
  • ACID compliance
  • Supports complex queries
  • Apps: Banking, inventory, ERP, student systems
NoSQL (JSON-Based)
  • Schema-less or flexible schema
  • Document-based storage
  • Horizontally scalable
  • Apps: Social, e-commerce, CMS, real-time analytics

APIs (JSON)

Why JSON is Preferred
  • Human-readable, lightweight
  • Language-independent
  • Native support in JavaScript
Real-World Applications
  • RESTful APIs
  • Mobile app backends
  • Web services
  • Cloud platforms

Configuration Files

Common Uses
  • Application settings
  • Server configuration
  • Environment variables
  • Deployment scripts
Real-World Examples
  • package.json in Node.js
  • docker-compose.yml in containerized systems
  • config.yaml in cloud deployments

Data Formats in AI & Machine Learning

Common Formats
  • CSV – simple datasets
  • JSON – metadata/annotations
  • HDF5 – large numerical datasets
  • TFRecord – TensorFlow training data
  • Pickle – Python object serialization (security risks)
Real-World Applications
  • Training deep learning models
  • Storing neural network weights
  • Managing labeled datasets
  • Exchanging models between systems

Choosing the Right Data Format

FactorConsideration
PerformanceBinary formats are faster
ReadabilityText formats are easier to debug
ScalabilityNoSQL formats scale better
CompatibilityJSON and Protobuf are cross-platform
SecurityBinary and encrypted formats offer better protection

Common Mistakes

  • Using JSON for large binary data instead of binary formats
  • Ignoring schema validation in flexible formats
  • Overusing text formats where performance is critical
  • Poor versioning of serialized data
  • Security risks from unsafe deserialization (e.g., Pickle)
  • Mixing incompatible formats across systems

Real-World Applications of Data Formats

Healthcare – structured DB + secure imaging
Gaming – binary assets
IoT – Protobuf, binary telemetry
Cloud – JSON APIs, YAML configs
